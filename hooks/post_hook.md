# ✅ 완료 후 훅 (Post-Hook) — 셀프 체크 리마인더

> **트리거:** AI 에이전트가 하나의 태스크를 완료했다고 판단했을 때, 루프를 종료하기 직전 **반드시** 이 문서의 모든 체크리스트를 수행해야 한다.
> 하나라도 누락 시 루프 종료를 선언할 수 없다.

## ✅ 필수 완료 후 체크리스트

### 1. 에러 및 회귀 테스트 점검
- [ ] 방금 작성하거나 수정한 코드에 **문법 에러(Syntax Error)**가 없는가?
- [ ] 기존 기능을 망가뜨리지 않았음을 확인했는가? (회귀 테스트)
- [ ] 테스트 코드가 존재한다면 실제로 실행하여 **모든 테스트가 통과**하는지 확인했는가?

> 💡 확신할 수 없다면, `skills/sprint_qa.md` 매뉴얼을 열고 스스로 터미널에서 컴파일/단위 테스트를 실행하여 에러 로그를 확인한 후 수정해라. **에러를 안고 루프를 끝내는 것은 절대 금지.**

### 2. 보안 취약점 및 하드코딩 점검
- [ ] API Key, 비밀번호, 인증 토큰 등을 코드에 **하드코딩하지 않았는가?** (`.env` 사용 권장)
- [ ] 민감한 데이터가 로그에 출력되지 않는가?
- [ ] SQL 인젝션, XSS 등 기본적인 보안 취약점이 없는가?

### 3. 아키텍처 및 도메인 규칙 준수
- [ ] 새로운 라이브러리/스택을 **무단으로 도입하지 않았는가?** (TRD Stack Contract 기준)
- [ ] `skills/domain_rules.md`(존재할 경우) 도메인 네이밍 규칙을 어기지 않았는가?
- [ ] 범위 외(Out of Scope) 기능을 슬쩍 추가하지 않았는가?

### 4. 외부 기억 장치(Context) 동기화 — ★ 가장 중요 ★
- [ ] **`context/3_checklist.md`:** 방금 해결한 작업을 `[x]`로 마킹했는가?
- [ ] **`context/2_context_note.md`:** 다음 루프의 AI가 알아야 할 모든 맥락을 기록했는가?
  - 발생한 에러와 해결 방법
  - 시도했으나 실패한 접근법
  - 다음 루프에서 필요한 변수 상태나 주의사항
  - 새로 발견된 오픈 이슈
- [ ] **`context/1_plan.md`:** 전체 계획이 수정되었다면 업데이트했는가?

> "모든 컨텍스트가 외부 기억 장치로 저장되었습니다. 루프를 종료합니다. **LOOP_DONE**"
