# 🛑 시작 전 훅 (Pre-Hook)

> **트리거:** 매 루프(작업 세션)가 시작될 때 AI가 코드를 한 줄이라도 짜기 전에 **반드시** 이 문서를 읽고 아래 체크리스트를 수행해야 한다.

## ✅ 필수 사전 체크리스트

### 1. 외부 기억 장치(Context) 로드
- [ ] `context/1_plan.md`를 읽고 **전체 프로젝트 목표와 아키텍처**를 숙지했는가?
- [ ] `context/2_context_note.md`를 읽고 **이전 루프의 결정사항, 오류 로그, 변수 상태**를 이해했는가?
- [ ] `context/3_checklist.md`를 읽고 **이번 루프에서 수행할 태스크**(가장 위의 `[ ]` 항목)를 확인했는가?

### 2. 스펙 주도 개발(SDD) 강제
- [ ] PRD 또는 기획 문서가 존재하는가? **기획이 없으면 코딩 금지.** 먼저 `skills/prd_builder.md`를 사용하여 PRD를 작성해야 한다.
- [ ] 이번 태스크의 범위(In Scope)가 명확한가? 범위가 모호하면 `context/2_context_note.md`에 `[ASSUMPTION]`을 기록하고 진행한다.

### 3. 도메인 주도 설계(DDD) 준수
- [ ] `skills/domain_rules.md`가 존재하면 반드시 읽고, 변수명/함수명/폴더명이 도메인 용어 사전과 일치하도록 코딩해야 한다.
- [ ] 기존 프로젝트의 코드 구조와 네이밍 패턴을 파악했는가?

### 4. 관련 스킬 매뉴얼 로드
- [ ] 이번 루프에서 수행할 작업에 해당하는 `skills/*.md` 매뉴얼을 확인했는가?
  - TRD 작성이면 → `skills/agent_trd.md`
  - 스프린트 구현이면 → `skills/implement_sprint.md`
  - QA이면 → `skills/sprint_qa.md`
  - 기타 해당 스킬 참조

### 5. 절대 금지사항 확인
- [ ] **기존 코드 파괴 금지** — 새 기능을 추가하더라도 기존에 잘 동작하던 핵심 로직이나 테스트를 훼손하지 않는다.
- [ ] **신규 스택 임의 도입 금지** — TRD/Stack Contract에 명시되지 않은 프레임워크/라이브러리를 무단 추가하지 않는다.
- [ ] **범위 외 작업 금지** — 체크리스트에 없는 작업은 BACKLOG에 기록하고 넘어간다.

---

> **⚠️ 위 사항이 모두 확인되지 않으면 작업을 시작하지 마십시오.**
> 모든 체크가 완료되면, `context/3_checklist.md`에서 확인한 태스크를 시작하십시오.
