# 🛑 시작 전 강제 훅 (Pre-Run Domain Check)

이 컨텐츠는 AI 에이전트가 코딩을 시작하기 전, 매 루프마다 **무조건 가장 먼저** 읽고 숙지해야 하는 도메인 제약사항 및 철학입니다.

## 🚨 철명(Iron Rules)
1. **스펙 주도 개발 (SDD) 준수:**
   코드를 단 한 줄이라도 작성하기 전, `context/1_plan.md`가 비어있거나 구체적이지 않다면 반드시 `skills/prd_builder.md`를 열어서 **완벽한 기획서(PRD/주문서)**를 먼저 작성해라. 기획이 모호한 상태에서 코딩을 시작하는 것은 금지된다.

2. **도메인 주도 설계 (DDD) 준수:**
   모든 변수명, 함수명, 파일 및 폴더명은 특정 비즈니스 도메인의 용어 사전(Ubiquitous Language)과 정확히 일치해야 한다. `skills/domain_rules.md` 가 존재한다면 반드시 해당 문서의 규칙을 따라라. 모듈간 결합도를 낮추고 응집도를 높여라.

3. **기존 코드 파괴 금지:**
   새로운 기능을 추가하더라도 절대 기존에 잘 동작하던 핵심 로직이나 테스트 케이스를 훼손하지 마라.

4. **외부 기억 장치 의존:**
   너의 이전 대화 기억은 존재하지 않는다. 오직 `context/` 폴더의 3개 문서만이 너의 기억이다. 반드시 이 문서들을 먼저 읽어라.
