# 3. 할 일 체크리스트 (Checklist)

> **💡 AI 에이전트 행동 지침:**
> 너는 이 문서에서 **아직 `[ ]`로 표시된 가장 위의 항목**을 찾아 이번 루프의 목표로 삼아야 해.
> 작업을 완전히 끝내고, 에러가 없는지(QA)까지 확인했다면 해당 항목을 `[x]` 처리해.
> 굵직한 태스크를 더 작은 단위로 쪼개야 한다면 하위 `[ ]` 항목들을 추가해도 좋아.

## Phase 0: PRD 작성 (스펙 주도 개발 강제)
- [ ] PRD 작성 또는 기존 PRD 검토 (`skills/prd_builder.md` 참고)

## Phase 1: 설계 (KICKOFF 또는 Enhancement)

### KICKOFF 트랙 (신규 프로젝트)
- [ ] 프로젝트 Kickoff 수행 (`skills/kickoff_project.md` 참고, `skills/0_kickoff_concerns.md` 필독)

### Enhancement 트랙 (MINOR/MAJOR)
- [ ] TRD / 아키텍처 설계 (`skills/agent_trd.md` 참고, `skills/0_process_concerns.md` 필독)
- [ ] API / 데이터 모델 설계 — MAJOR 필수, MINOR 스킵 가능 (`skills/agent_api.md` 참고)
- [ ] 실행 계획 / 백로그 초안 — MAJOR 필수, MINOR 스킵 가능 (`skills/agent_plan.md` 참고)

## Phase 2: 문서 통합 및 백로그 확정
- [ ] 문서 통합 (Final Merge) (`skills/final_merge.md` 참고)
- [ ] 백로그 확정 — MAJOR 필수 (`skills/final_backlog.md` 참고)

## Phase 3: 스프린트 루프 (구현 → QA 반복)
- [ ] Sprint 1 구현 (`skills/implement_sprint.md` 참고)
